# 默认提示词已集成到系统初始化

## 📦 打包版本说明

从当前版本开始，所有爆款写作提示词已经**直接集成到系统初始化代码中**，无需手动导入。

### ✅ 已集成的提示词

#### 写作要求（4 个）
1. **写作要求** - 通用爆款网文写作指南（带题材字段）
2. **爆款知乎风，一次 1w，生成停止请使用续写继续写** - 知乎风格万字短篇生成器
3. **女频写作 - 情感共鸣** - 番茄/飞卢女频爆款写作指南
4. **爆款写作 -【番茄风】** - 番茄平台风格写作指南

#### 写作风格（1 个）
1. **乐子文风格** - 接地气、反套路的轻松写作风格

## 🔧 工作原理

### 首次运行（新安装）
当用户首次运行网文助手系统时：
1. 系统自动初始化数据库
2. 检测到没有默认提示词
3. 自动导入全部 5 个预设提示词
4. 用户可以直接使用

### 后续运行
- 系统检测到提示词已存在
- 跳过导入，避免重复
- 用户可以正常使用或修改这些提示词

### 打包版本
无论使用何种打包方式（pkg、便携版等），默认提示词都会：
- ✅ 包含在程序代码中
- ✅ 首次运行时自动导入
- ✅ 无需额外的导入步骤
- ✅ 无需外部 JSON 文件

## 📝 技术实现

### 修改文件
- `server/database/db.js` - 添加了 `importDefaultPrompts()` 函数

### 核心逻辑
```javascript
// 在数据库初始化后自动执行
const importDefaultPrompts = () => {
  // 检查是否已存在
  const existingPrompt = checkPrompt.get('写作要求', '写作要求');
  
  if (existingPrompt) {
    console.log('默认提示词已存在，跳过导入');
    return;
  }

  // 导入 5 个默认提示词
  // ...
};

// 初始化数据库
initDatabase();

// 导入默认提示词
importDefaultPrompts();
```

## 🎯 使用说明

### 新用户
1. 安装/解压网文助手
2. 首次启动系统
3. 在"提示词管理"模块查看预设提示词
4. 直接使用或根据需要修改

### 老用户（已安装）
如果已经安装并使用过系统，可以选择：

**方式 1：重新初始化（推荐）**
```bash
# 删除旧数据库（会丢失现有提示词）
删除 server/database/novel.db

# 重新启动系统，会自动导入默认提示词
```

**方式 2：手动导入**
```bash
# 使用导入脚本
node import-prompts.js
```

**方式 3：批量更新**
如果数据库中已有同名提示词，运行导入脚本会自动更新内容。

## ⚠️ 注意事项

1. **不要随意删除数据库** - 除非你想重置所有数据
2. **提示词可以修改** - 用户可以在系统中修改这些提示词
3. **修改不会被覆盖** - 已存在的提示词不会被自动覆盖
4. **打包后无需额外操作** - 所有提示词都已内置

## 🚀 优势

相比之前的方式，这种集成方案有以下优势：

1. ✅ **开箱即用** - 无需手动导入
2. ✅ **打包友好** - 提示词内置在代码中
3. ✅ **版本控制** - 提示词随代码一起管理
4. ✅ **自动检测** - 避免重复导入
5. ✅ **用户友好** - 新用户零配置

## 📞 问题反馈

如果打包后仍然没有提示词，请检查：
1. 系统日志中是否有"导入默认提示词"的记录
2. 数据库文件是否正确生成
3. 提示词管理模块中是否有数据
